## Objective

Build article detail view that displays full article content with all formatting, images, and embedded videos preserved, along with complete metadata.

## Context

useArticles hook manages articles with TipTap HTML content. We need a view that renders this HTML safely while preserving all rich text formatting, images, and YouTube embeds from Phase 3.

## Tasks

<task id="1">
**Create ArticleDetail component**

Create `src/components/Blog/ArticleDetail.tsx`:
- Accept article prop (Article type)
- Accept onBack callback to return to blog list
- Display full article with Skyblog styling:
  - Title as h1 with pink underline (#FF1493)
  - Metadata line: "Publié le DD/MM/YYYY à HH:MM"
  - Full content rendered with dangerouslySetInnerHTML (content already sanitized by TipTap)
  - Tags section with clickable tag links (green color #00FF00)
  - "← Retour au blog" button at top
  - Bottom metadata: "X kiffs | X commentaires" (placeholders for Phase 5)
- Style matches ArticlePreview but full-width content
- Ensure TipTap formatting renders correctly (colors, fonts, bold, italic, underline)
- Ensure images display inline
- Ensure YouTube iframes render and play

Success: Article displays with all formatting and media preserved
</task>

<task id="2">
**Add article selection state to App.tsx**

In `src/App.tsx`:
- Add state: `selectedArticleId: string | null`
- Add handler: `handleSelectArticle(id: string)` → sets selectedArticleId
- Add handler: `handleBackToList()` → clears selectedArticleId
- Modify content prop when view === 'blog':
  - If selectedArticleId: render ArticleDetail with article from getArticle(selectedArticleId)
  - If no selectedArticleId: render BlogHome (existing)
- Wire onArticleClick in BlogHome to handleSelectArticle
- Wire onBack in ArticleDetail to handleBackToList

Success: Clicking article navigates to detail view, back button returns to list
</task>

<task id="3">
**Update ArticlePreview click handling**

In `src/components/Blog/ArticlePreview.tsx`:
- Make entire preview clickable (onClick calls onArticleClick with article.id)
- Or make "Lire la suite" link call onArticleClick
- Add cursor: pointer style to indicate clickability
- Hover effect: slight background color change (#222)

Success: Clicking article preview opens detail view
</task>
