## Objective

Enable visitors to filter articles by clicking on tags, showing only articles with that specific tag.

## Context

Articles have tags as string arrays. Need to add filtering state and UI to show "filtered by tag" view with clear indicator and ability to clear filter.

## Tasks

<task id="1">
**Add tag filtering state to App.tsx**

In `src/App.tsx`:
- Add state: `selectedTag: string | null`
- Add handler: `handleTagClick(tag: string)` → sets selectedTag, clears selectedArticleId
- Add handler: `handleClearTagFilter()` → sets selectedTag to null
- Pass selectedTag and handlers to BlogHome and ArticleDetail
- Filter articles in BlogHome based on selectedTag:
  - If selectedTag: articles.filter(a => a.tags.includes(selectedTag))
  - If null: show all published articles

Success: Tag filter state managed at App level
</task>

<task id="2">
**Update BlogHome to show filter indicator**

In `src/components/Blog/BlogHome.tsx`:
- Accept selectedTag and onClearTagFilter props
- When selectedTag is set, show filter banner above articles:
  - Message: "Articles avec le tag: [tag]"
  - Clear button: "✕ Voir tous les articles"
  - Style with green tag color (#00FF00)
- Empty state when filtered and no results: "Aucun article avec ce tag."
- Make tags in ArticlePreview clickable: call onTagClick(tag)
- Style tag links with hover effect

Success: Tag filtering UI integrated in BlogHome
</task>

<task id="3">
**Make tags clickable in ArticleDetail**

In `src/components/Blog/ArticleDetail.tsx`:
- Accept onTagClick prop
- Make each tag a clickable link
- onClick: call onTagClick(tag) → navigates back to filtered blog list
- Style tags as clickable with green color and hover effect
- Add cursor: pointer

Success: Tags clickable in both list and detail views
</task>
