## Objective

Build blog homepage that displays a list of published articles with titles, preview text, metadata (date, tags), and profile information visible in the sidebar.

## Context

Currently the 'blog' view in App.tsx shows placeholder content. We need to replace this with real published articles from useArticles hook. Only published articles (isDraft: false) should be visible. Profile already displays in left sidebar via SkyblogLayout.

## Tasks

<task id="1">
**Create ArticlePreview component**

Create `src/components/Blog/ArticlePreview.tsx`:
- Accept article prop (Article type)
- Display article in Skyblog 2006 style:
  - Title as h1 with pink underline (#FF1493)
  - Date formatted as DD/MM/YYYY à HH:MM
  - Preview text: first 200 characters of content (strip HTML tags)
  - Tags as clickable links (prepare onClick prop for future filtering)
  - "Lire la suite" (Read more) link
  - Metadata line: "Publié le [date] | X kiffs | Commenter"
- Style with #1a1a1a background, white text, Verdana font
- Add bottom margin for spacing between articles

Success: Article preview displays with authentic Skyblog aesthetic
</task>

<task id="2">
**Create BlogHome component**

Create `src/components/Blog/BlogHome.tsx`:
- Accept articles array and profile from props
- Filter to only published articles (isDraft === false)
- Sort by createdAt descending (newest first)
- Map each to ArticlePreview component
- Pass onTagClick and onArticleClick callbacks as props
- Empty state when no published articles: "Aucun article publié pour le moment."
- Wrapper div with className="blog-content"

Success: BlogHome displays list of published articles, newest first
</task>

<task id="3">
**Integrate BlogHome into App.tsx**

In `src/App.tsx`:
- Import BlogHome component
- In content prop, when view === 'blog': render BlogHome instead of placeholder
- Pass published articles (filtered from useArticles)
- Pass profile from useProfile for potential use
- Pass onTagClick handler (placeholder for now, Phase 4 Plan 2)
- Pass onArticleClick handler (placeholder for now, Phase 4 Plan 2)
- Keep placeholder kiffs/comments counts (will be real data in Phase 5)

Success: Blog view shows real published articles with metadata
</task>
