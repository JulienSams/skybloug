## Objective

Build interactive profile components that allow users to view and edit their profile, replacing the hardcoded profile in App.tsx.

## Context

Plan 02-01 created the profile data structure and useProfile hook. Now we build the UI: a display component that shows the profile and an edit component with form fields and photo upload.

## Tasks

<task id="1">
**Create ProfileDisplay component**

Create `src/components/Profile/ProfileDisplay.tsx`:
- Accept profile data and onEdit callback as props
- Render profile photo (120x120, centered, white border, #FF1493 background if no photo)
- Display name as h2, centered, white color
- Display bio, age, location in same style as current App.tsx (gray text, 11px font)
- Add "Modifier" button that triggers onEdit callback
- Match exact styling from current hardcoded profile

Success: Component displays profile data matching Phase 1 aesthetic
</task>

<task id="2">
**Create ProfileEdit component**

Create `src/components/Profile/ProfileEdit.tsx`:
- Accept profile data, update functions, and onSave/onCancel callbacks as props
- Photo upload section: clickable area that opens file picker, preview of current photo
- Form fields for: name (text input), bio (textarea), age (number input), location (text input)
- Save and Cancel buttons
- Apply Skyblog styling: dark background, white/gray text, #FF1493 accents
- Handle photo file selection and pass to useProfile's handlePhotoUpload

Success: Form allows editing all 5 fields with proper styling and callbacks
</task>

<task id="3">
**Integrate ProfileDisplay into App.tsx**

In `src/App.tsx`:
- Import useProfile hook and ProfileDisplay/ProfileEdit components
- Initialize profile with useProfile()
- Replace hardcoded leftSidebar JSX with ProfileDisplay component
- Add state to toggle between display and edit modes
- When edit mode active, render ProfileEdit instead of ProfileDisplay
- Pass profile data and update functions to components

Success: App.tsx renders dynamic profile that can switch to edit mode
</task>

<task id="4">
**Implement edit mode toggle**

In `src/App.tsx`:
- Add useState for editMode boolean
- ProfileDisplay's onEdit sets editMode to true
- ProfileEdit's onSave updates profile and sets editMode to false
- ProfileEdit's onCancel sets editMode to false without saving
- Updates display immediately in sidebar after save

Success: User can click edit, modify fields, save or cancel, see updates immediately
</task>
