## Objective

Build article management interface where users can view their article list, edit existing articles, and delete articles with confirmation.

## Context

User wants simple list with status badges, "Manage Articles" button in right sidebar, delete confirmation dialog. Editor replaces center content when editing. Following Phase 2's edit mode toggle pattern.

## Tasks

<task id="1">
**Create ArticleList component**

Create `src/components/Articles/ArticleList.tsx`:
- Accept articles array, onEdit (articleId), onDelete (articleId), onNewArticle callbacks
- Render "New Article" button at top with #FF1493 Skyblog styling
- Render vertical list of articles showing:
  - Title
  - Status badge (Draft in gray, Published in pink #FF1493)
  - Date (createdAt formatted as DD/MM/YYYY)
  - Edit button (loads article in editor)
  - Delete button (shows confirmation)
- Empty state when no articles: friendly message like "No articles yet. Click 'New Article' to start writing!"
- Style with Skyblog dark theme (#1a1a1a background, white text, section dividers)

Success: Article list displays all articles with actions and empty state
</task>

<task id="2">
**Add delete confirmation dialog**

In ArticleList component:
- When Delete clicked, show confirmation before calling onDelete
- Use window.confirm() or create simple dialog component
- Message: "Are you sure you want to delete '[article title]'?"
- Cancel button - closes dialog without action
- Delete button - calls onDelete(articleId) and closes dialog
- Style dialog with Skyblog aesthetic if custom component

Success: Delete requires confirmation, prevents accidental deletion
</task>

<task id="3">
**Integrate ArticleList into App.tsx**

In `src/App.tsx`:
- Import useArticles hook and ArticleList, ArticleEditor components
- Add state for current view: 'blog' | 'articles' | 'editor'
- Add state for editingArticleId (null | string) to track which article is being edited
- When view === 'articles': render ArticleList in content area
- When view === 'editor': render ArticleEditor in content area
- When view === 'blog': show existing blog content (placeholder articles)
- Wire up callbacks:
  - onNewArticle: set view to 'editor', editingArticleId to null
  - onEdit: set view to 'editor', editingArticleId to clicked article id
  - onDelete: call deleteArticle from hook
  - editor onSave: call createArticle or updateArticle based on editingArticleId, return to 'articles' view
  - editor onCancel: return to 'articles' view

Success: App switches between blog view, article list, and editor correctly
</task>

<task id="4">
**Add "Manage Articles" button to right sidebar**

In `src/App.tsx` rightSidebar section:
- Add "Manage Articles" button styled like Skyblog button (#FF1493 background, white text)
- Place it in a new section at top of right sidebar, above Info section
- Button onClick: set view to 'articles'
- When in articles or editor view, maybe show "Back to Blog" button

Success: Users can navigate to article management from sidebar
</task>
