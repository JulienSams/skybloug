## Objective

Integrate TipTap rich text editor with classic 2006 Skyblog toolbar aesthetic, supporting all formatting requirements and draft/publish workflow.

## Context

User wants TipTap library with nostalgic Skyblog styling (colorful buttons, #FF1493 pink accents, Verdana fonts). Extended formatting controls include bold, italic, underline, colors, fonts, size, lists, links, and alignment. Editor replaces center content area.

## Tasks

<task id="1">
**Install TipTap dependencies**

Add to package.json and install:
- @tiptap/react
- @tiptap/starter-kit
- @tiptap/extension-text-style
- @tiptap/extension-color
- @tiptap/extension-font-family
- @tiptap/extension-text-align
- @tiptap/extension-link
- @tiptap/extension-image
- @tiptap/extension-youtube

Run npm install after updating package.json

Success: TipTap packages installed, app still runs
</task>

<task id="2">
**Create EditorToolbar component**

Create `src/components/Editor/EditorToolbar.tsx`:
- Accept TipTap editor instance as prop
- Toolbar buttons for: Bold, Italic, Underline, Text Color picker, Font Family dropdown, Font Size dropdown, Background Color picker, Bullet List, Numbered List, Link, Text Alignment (left/center/right)
- Style with 2006 Skyblog aesthetic: horizontal bar, colorful icon buttons, #FF1493 pink for active states, Verdana font, bright colors on dark background
- Use inline styles matching Phase 1-2 patterns
- Button states: active styling when format is applied at cursor

Success: Toolbar component renders all formatting controls with Skyblog styling
</task>

<task id="3">
**Create ArticleEditor component**

Create `src/components/Editor/ArticleEditor.tsx`:
- Accept onSave (with status: 'draft' | 'published'), onCancel, initialArticle (optional for editing) as props
- Initialize TipTap editor with all extensions (StarterKit, TextStyle, Color, FontFamily, TextAlign, Link, Image, YouTube)
- Render EditorToolbar with editor instance
- Title input field above editor
- TipTap EditorContent component for rich text area
- Tags input field below editor (text input, comma-separated, show as chips)
- Two buttons: "Save as Draft" and "Publish" (both call onSave with different status)
- Cancel button
- Style with Skyblog dark theme (#1a1a1a background, white text, #FF1493 accents)

Success: Complete editor interface with all required controls
</task>

<task id="4">
**Handle editor content serialization**

In ArticleEditor:
- Convert TipTap editor content to HTML string when saving (editor.getHTML())
- Load initial content from HTML when editing existing article (editor.commands.setContent())
- Handle empty state gracefully
- Parse tags from comma/space-separated input into string array

Success: Editor can save and load article content correctly
</task>
