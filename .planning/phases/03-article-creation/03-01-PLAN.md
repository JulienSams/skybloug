## Objective

Create the article data structure and state management foundation that will power the entire article creation and management system.

## Context

Phase 2 established patterns for TypeScript types and custom hooks (Profile.ts, useProfile.ts). This plan follows the same approach for articles, supporting all 11 requirements with draft/published status, tags, and content storage.

## Tasks

<task id="1">
**Create Article type definition**

Create `src/types/Article.ts` with:
- Interface defining article fields: id (string), title (string), content (string - will hold TipTap HTML), status ('draft' | 'published'), tags (string[]), createdAt (Date), updatedAt (Date)
- Helper type for article creation (Omit<Article, 'id' | 'createdAt' | 'updatedAt'>)
- Export both for use throughout the app

Success: Type file exists with complete interface covering all article data needs
</task>

<task id="2">
**Create useArticles hook**

Create `src/hooks/useArticles.ts` with:
- useState to hold articles array (initialized empty)
- createArticle function (accepts NewArticle, generates ID/timestamps, adds to array)
- updateArticle function (accepts id and partial updates, updates article in array)
- deleteArticle function (accepts id, removes from array)
- getArticle function (accepts id, returns single article or undefined)
- Return object with articles array and all CRUD functions
- Use TypeScript for full type safety

Success: Hook provides complete state management for article operations
</task>

<task id="3">
**Add article filtering helpers**

In `src/hooks/useArticles.ts`, add computed values:
- drafts - filtered list of articles where status === 'draft'
- published - filtered list of articles where status === 'published'
- Return these alongside articles array for easy access

Success: Hook provides convenient draft/published filtering
</task>
