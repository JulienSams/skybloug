## Objective

Implement API endpoints for profile management (single-user) and article comments.

## Context

Single-user blog with one profile. Profile endpoints get/update the single profile record. Comments are associated with articles via articleId foreign key.

## Tasks

<task id="1">
**Create profile controller and router**

Create `server/src/controllers/profileController.ts`:
- **getProfile**: GET handler
  - Query first profile record
  - If none exists, create default profile
  - Return profile
- **updateProfile**: PUT handler
  - Accept: name, bio, age, location, photoUrl
  - Update or create profile (upsert)
  - Return updated profile

Create `server/src/routes/profile.ts`:
- GET /api/profile → getProfile
- PUT /api/profile → updateProfile
- Export router

Success: Profile API with get/update operations
</task>

<task id="2">
**Create comments controller and router**

Create `server/src/controllers/commentController.ts`:
- **getComments**: GET handler for article
  - Accept articleId param
  - Query comments where articleId matches
  - Order by createdAt asc (oldest first)
  - Return comments array
- **createComment**: POST handler
  - Accept: articleId, authorName, content
  - Validate article exists (return 404 if not)
  - Create comment
  - Return created comment

Create `server/src/routes/comments.ts`:
- GET /api/articles/:articleId/comments → getComments
- POST /api/articles/:articleId/comments → createComment
- Export router

Success: Comments API with get/create operations
</task>

<task id="3">
**Integrate routers into server**

In `server/src/index.ts`:
- Import profile and comments routers
- Use routers:
  - app.use('/api', profileRouter)
  - app.use('/api', commentsRouter)

Test with curl or Postman:
- GET /api/profile (creates default if missing)
- PUT /api/profile with updated data
- POST /api/articles/:id/comments
- GET /api/articles/:id/comments

Success: Profile and comments APIs fully functional
</task>
