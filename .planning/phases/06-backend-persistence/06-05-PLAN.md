## Objective

Replace client-side state with API calls in all hooks. Update image upload to use server endpoint. Add loading states and error handling.

## Context

Backend API complete with all endpoints. Frontend currently uses useState for all data. Need to replace with fetch calls to backend API and handle async operations.

## Tasks

<task id="1">
**Create API service**

Create `src/services/api.ts`:
- Base URL from env: VITE_API_URL (default http://localhost:3000/api)
- Helper functions for all endpoints:
  - Articles: getArticles, getArticle, createArticle, updateArticle, deleteArticle
  - Profile: getProfile, updateProfile
  - Comments: getComments, createComment
  - Images: uploadImage
- Add error handling and JSON parsing
- Export all functions

Create `.env`:
- VITE_API_URL=http://localhost:3000/api

Success: API service ready for hook integration
</task>

<task id="2">
**Update useArticles hook**

In `src/hooks/useArticles.ts`:
- Replace useState with API calls
- Add loading state
- useEffect to fetch articles on mount
- createArticle → POST to API
- updateArticle → PUT to API
- deleteArticle → DELETE to API
- Optimistic updates for better UX
- Error handling with console.error

Success: Articles persist across page refreshes
</task>

<task id="3">
**Update useProfile and useComments hooks**

In `src/hooks/useProfile.ts`:
- Fetch profile on mount (GET /api/profile)
- Update functions → PUT /api/profile
- Update photoUrl field to use server URL
- Handle base64 → upload conversion

In `src/hooks/useComments.ts`:
- Remove global state (comments per article from API)
- getCommentsForArticle → GET from API
- addComment → POST to API
- Return loading state

Success: Profile and comments persist
</task>

<task id="4">
**Update image upload in EditorToolbar**

In `src/components/Editor/EditorToolbar.tsx`:
- Change image upload to use API
- Create FormData with file
- POST to /api/images
- Use returned URL in TipTap editor
- Replace base64 with server URL
- Add upload progress/loading indicator

Success: Images uploaded to server and embedded via URL
</task>
