## Objective

Set up Node.js + Express + TypeScript backend server with PostgreSQL database. Define database schema using Prisma ORM for articles, profiles, comments, and tags.

## Context

Frontend complete with all features working in component state. Need backend API to persist data across sessions and devices. Using Prisma for type-safe database access and migrations.

## Tasks

<task id="1">
**Initialize backend project**

Create `server/` directory and initialize Node.js project:
- Run `npm init -y` in server directory
- Install dependencies:
  - express, cors, dotenv
  - @prisma/client, prisma (dev)
  - typescript, ts-node, @types/node, @types/express, @types/cors (dev)
  - nodemon (dev)
- Create tsconfig.json with:
  - target: ES2020
  - module: commonjs
  - outDir: dist
  - rootDir: src
- Add scripts to package.json:
  - dev: nodemon src/index.ts
  - build: tsc
  - start: node dist/index.js
- Create .env.example with DATABASE_URL, PORT

Success: Backend project initialized with TypeScript
</task>

<task id="2">
**Set up Express server**

Create `server/src/index.ts`:
- Import express, cors, dotenv
- Configure CORS for frontend (http://localhost:5174)
- Create Express app
- Add JSON body parser middleware
- Add health check endpoint: GET /api/health
- Start server on PORT (default 3000)
- Log startup message with port

Success: Express server runs and responds to health check
</task>

<task id="3">
**Define Prisma schema**

Create `server/prisma/schema.prisma`:
- Configure PostgreSQL datasource and Prisma client generator
- Define models:
  - **Profile**: id, name, bio, age, location, photoUrl, createdAt, updatedAt
  - **Article**: id, title, content (HTML), isDraft, createdAt, updatedAt, tags (relation)
  - **Tag**: id, name, articles (relation many-to-many)
  - **Comment**: id, articleId (FK), authorName, content, createdAt
- Use UUID for IDs
- Set up Tag ↔ Article many-to-many with ArticleTag join table
- Add indexes on articleId for comments, name for tags

Success: Schema defined with all models and relations
</task>

<task id="4">
**Initialize database and run migrations**

In server directory:
- Run `npx prisma init` (if not done)
- Set DATABASE_URL in .env to local PostgreSQL
- Run `npx prisma migrate dev --name init` to create initial migration
- Run `npx prisma generate` to generate Prisma Client
- Verify database created with all tables

Add to README or docs:
- PostgreSQL installation instructions
- Database setup commands
- Migration commands

Success: Database created with all tables, Prisma Client generated
</task>
