## Objective

Implement image upload endpoint with multer middleware and serve uploaded images via static file serving.

## Context

Currently images are base64 encoded in article content. Need to upload images to server filesystem and return URLs that can be embedded in articles. TipTap Image extension will use upload endpoint.

## Tasks

<task id="1">
**Set up multer for file uploads**

Install multer:
- Run `npm install multer @types/multer`

Create `server/src/middleware/upload.ts`:
- Import multer
- Configure storage:
  - destination: './uploads'
  - filename: timestamp + random string + original extension
- File filter: accept only image types (jpg, jpeg, png, gif, webp)
- Size limit: 5MB per image
- Export configured multer middleware

Create uploads directory:
- Create `server/uploads/` directory
- Add to .gitignore (images not committed to repo)

Success: Multer middleware configured for image uploads
</task>

<task id="2">
**Create image upload endpoint**

Create `server/src/controllers/imageController.ts`:
- **uploadImage**: POST handler
  - Multer middleware handles file upload
  - Generate public URL: `/uploads/${filename}`
  - Return JSON: { url: publicUrl }
  - Handle errors (file too large, wrong type)

Create `server/src/routes/images.ts`:
- POST /api/images → uploadImage (with multer middleware)
- Export router

Success: Image upload endpoint accepts images and returns URLs
</task>

<task id="3">
**Serve uploaded images as static files**

In `server/src/index.ts`:
- Import path
- Add static file serving: app.use('/uploads', express.static('uploads'))
- Import and use images router
- Ensure CORS allows image requests

Test:
- Upload image via POST /api/images
- Access returned URL in browser
- Verify image displays

Success: Images uploaded and accessible via URL
</task>
