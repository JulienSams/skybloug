## Objective

Implement RESTful API endpoints for articles with full CRUD operations, draft support, and tag management.

## Context

Database schema in place with Article, Tag, and ArticleTag models. Need API endpoints to create, read, update, and delete articles. Tags are created/associated automatically when saving articles.

## Tasks

<task id="1">
**Create article controller**

Create `server/src/controllers/articleController.ts`:
- Import PrismaClient
- **createArticle**: POST handler
  - Accept: title, content, isDraft, tags (array)
  - Create/connect tags (upsert by name)
  - Create article with tag relations
  - Return created article with tags
- **getArticles**: GET handler
  - Accept query param: draft (boolean)
  - Filter by isDraft if specified
  - Include tags in response
  - Order by createdAt desc
  - Return articles array
- **getArticle**: GET by ID handler
  - Accept id param
  - Include tags in response
  - Return 404 if not found
- **updateArticle**: PUT handler
  - Accept id, title, content, isDraft, tags
  - Update article and reconnect tags
  - Return updated article with tags
- **deleteArticle**: DELETE handler
  - Accept id param
  - Delete article (cascade to ArticleTag)
  - Return 204 No Content

Success: All CRUD operations implemented
</task>

<task id="2">
**Create articles router**

Create `server/src/routes/articles.ts`:
- Import express Router
- Import article controller functions
- Define routes:
  - POST /api/articles → createArticle
  - GET /api/articles → getArticles
  - GET /api/articles/:id → getArticle
  - PUT /api/articles/:id → updateArticle
  - DELETE /api/articles/:id → deleteArticle
- Export router

Success: Router configured with all article endpoints
</task>

<task id="3">
**Integrate articles router into server**

In `server/src/index.ts`:
- Import articles router
- Use router: app.use('/api', articlesRouter)
- Ensure routes accessible at /api/articles/*

Test with curl or Postman:
- Create article with tags
- Get all articles
- Get single article
- Update article
- Delete article

Success: Articles API fully functional
</task>
